# 🌲 在森林那邊 - 庫存系統操作手冊 (SOP) V3.1

**版本：** V3.1 (2026/01/29)
**適用對象：** 全體員工

---

## 🚀 一、每日出貨流程 (Daily Picking)

由於系統升級為 V3.1 潔淨版，現在**蝦皮與官網**皆統一使用「手動匯入」方式，確保資料準確。

### 步驟 1：準備訂單資料
* **【蝦皮訂單】**：
    1.  進入蝦皮賣家中心 -> 「待出貨」。
    2.  `Ctrl + A` 全選網頁文字 -> 複製。
    3.  貼上到 Google Sheet `[00_數據暫存區]` 的 **A 欄** (A2 開始)。
* **【官網訂單】**：
    1.  進入 WordPress 後台 -> WooCommerce -> 訂單 (狀態選「處理中」)。
    2.  選取訂單列表範圍 (包含商品、總額、狀態) -> 複製。
    3.  貼上到 Google Sheet `[00_數據暫存區]` 的 **C 欄** (C2 開始)。

### 步驟 2：產生撿貨單
1.  開啟 **庫存系統 Web App** (手機或電腦皆可)。
2.  切換到 **「撿貨」** 分頁。
3.  按下黃色的 **「📄 產生撿貨單」** 按鈕。
4.  系統會自動合併兩邊的訂單，並扣除庫存。

### 步驟 3：開始撿貨
1.  等待系統跑完，畫面會顯示 **「撿貨清單」**。
2.  同時 LINE 群組會跳出 **「今日撿貨通知」**。
3.  **注意**：如果 LINE 跳出 **「🔥 嚴重超賣警告」**，請立即通知主管處理。

## 📥 二、入庫與盤點 (Inventory)

### 生產入庫
* 當您完成產品製作 (如：灌好蠟燭、做好護木油) 時：
    1.  Web App -> **「入庫」** 分頁。
    2.  選擇商品 -> 輸入數量 -> **確認入庫**。

### 庫存盤點
* 發現架上數量跟系統不一樣？
    1.  Web App -> **「庫存」** 分頁。
    2.  點擊該商品的數字。
    3.  輸入 **「目前實際看到的數量」** (系統會自動計算差額)。

---

## 📅 三、每週補貨 (Weekly Restock)

* **時間**：每週一早上 09:00。
* **動作**：系統會自動檢查所有庫存。
* **通知**：若有低於安全水位的商品，LINE 群組會跳出 **「補貨提醒清單」**，請採購人員依照清單叫貨。

## 💡 常見問題 (FAQ)

* **Q: 匯入錯了怎麼辦？(例如貼到昨天的)**
    * A: 別緊張！按 Web App 上紅色的 **「↩️ 回復上一步」** 按鈕，系統會自動刪除剛剛的紀錄並把庫存加回去。
* **Q: 官網訂單沒抓到？**
    * A: 請確認是否有貼在 `[00_數據暫存區]` 的 **C 欄**。
* **Q: LINE 沒有跳通知？**
    * A: 只要 Web App 畫面有顯示撿貨單就是成功了。LINE 通知偶爾會有延遲。
- **Q: 蝦皮訂單貼上後無法辨識？**
    - A: 請確認是否有複製到完整的商品名稱與數量。如果蝦皮改版導致格式跑掉，請通知主管更新 `ShopeeParser`。
* **Q: 按了產生撿貨單，但庫存沒變？**
    * A: 請確認 App 是否顯示「✅ 庫存儀表板重算完成」。若有錯誤訊息，請截圖回報。
* **Q: 為什麼撿貨單上有些沒有物流單號？**
    * A: 這是新功能。如果是「蝦皮店到店」且只有「1個大罐」，系統會自動隱藏單號，讓畫面更乾淨。
* **Q: 庫存好像沒扣對？**
    * A: 請執行一次「產生撿貨單」或「刷新」，查看彈窗下方的 `🔍 [查帳]` 訊息。系統現在有「強力去汙」功能，能過濾掉看不見的空白字元，通常刷新後就會正常。
