# 🌲 在森林那邊 - 庫存系統操作手冊 (SOP)

**版本：** V2.3 (2026/01/23)
**適用對象：** 全體員工

---

## 🚀 開始使用前 (第一次設定)
1.  開啟 Web App。
2.  在右上角的選單中，選擇 **「你的名字」** (例如：員工A)。
3.  系統會記住你是誰，之後的操作紀錄都會掛在你的名下。

---

## 📦 角色一：包貨人員 (每日出貨)
**職責：** 早上上班時，確認今日訂單，進行撿貨、包裝、出貨。

1.  **刷新清單**：點擊「撿貨」頁面右上角的 **「刷新」**。
2.  **一鍵回報**：點擊 **「複製」** 按鈕，將整理好的清單貼到 Line 群組。
3.  **撿貨包裝**：依照清單作業。
4.  **結單清場**：
    * 當所有包裹都處理完畢。
    * 滑到最下方按下 **「全部出貨完成」**。
    * *注意：此動作無法復原，請確認真的包完了！*

---

## 📊 角色二：全體人員 (庫存查詢與盤點)
**職責：** 隨時確認架上庫存，發現數量不對立刻修正。

* **燈號看哪裡？**
    * 🔴 **需補貨**：低於安全水線，請優先處理！
    * ⚪ **已下架**：灰色卡片，代表不賣了。
* **數量不對怎麼辦？ (盤點修正)**
    1.  在庫存頁面，直接 **點擊該商品的數字**。
    2.  輸入 **「目前架上實際有多少」** (例如算出 47 個，就輸入 47)。
    3.  按確定，右下角會出現綠色通知：「修正完成」。

---

## 🏭 角色三：生產人員 (入庫作業)
**職責：** 成品製作完成、原料/包材到貨時登錄。

1.  切換到 **「入庫」** 分頁。
2.  **選擇項目**：選單已分類 (商品/原料/包材)，找不到請確認是否被歸類在「組合」(組合不可入庫)。
3.  **輸入數量**：輸入本次新增的數量。
4.  按下 **「確認入庫」**。

---

## 👤 角色四：管理員 (系統維護)

### A. 每日訂單匯入 (手機/電腦皆可)
1.  **準備資料**：將蝦皮/官網訂單複製貼上到 Google Sheet `[00_數據暫存區]`。
2.  **執行匯入**：
    *   **方法一 (推薦)**：開啟 Web App，點擊撿貨頁面上方的黃色 **「⚡ 匯入」** 按鈕。
    *   **方法二 (傳統)**：在電腦版點擊選單「擴充功能」 -> Apps Script -> 執行 `generateDailyPickingList`。
3.  **完成**：系統會自動計算扣除庫存，並產生合併後的撿貨單。

### B. 自動封存機制
* 系統會 **每月自動** 檢查銷售資料。
* 若 `[03_銷售數據池]` 超過 2000 筆，舊資料會自動移到 `[Backup_Sales]`。
* **你不需做任何動作**，若覺得系統變慢，可手動檢查備份表是否正常產出。

### 🚨 常見問題
| 問題 | 原因 | 解法 |
| :--- | :--- | :--- |
| **庫存一片白** | 後端數據未更新 | 請管理員執行 `InventoryManager.refreshDashboard()`。 |
| **顯示 "系統忙碌中"** | 多人同時操作 | 這是保護機制。請等 2-3 秒後再試一次即可。 |
| **入庫選單找不到某商品** | 商品是「組合包」 | 系統設定「組合」不可入庫。請拆解成單品入庫。 |
